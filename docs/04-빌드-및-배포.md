# 빌드 및 배포 가이드

## 사전 요구사항
- Flutter SDK 3.38.7+
- Android SDK (APK 빌드 시)
- Git

## 의존성 설치

```bash
flutter pub get
```

## 코드 생성 (Drift)

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

---

## Android APK 빌드

### Release APK (설치용, 추천)
```bash
flutter build apk --release
```
- 출력: `build/app/outputs/flutter-apk/app-release.apk` (~50MB)

### Debug APK (개발용)
```bash
flutter build apk --debug
```
- 출력: `build/app/outputs/flutter-apk/app-debug.apk` (~141MB)

### APK 설치 방법
1. Google Drive, 카카오톡 등으로 APK를 폰에 전송
2. 파일 열기 → "출처를 알 수 없는 앱 설치 허용"
3. 설치 완료

## Web 빌드

```bash
flutter build web --release
```
- 출력: `build/web/` 디렉토리

### 웹 DB 의존성 (sql.js)
`web/index.html`에 sql.js CDN이 포함되어 있음:
```html
<script src="https://cdn.jsdelivr.net/npm/sql.js@1.8.0/dist/sql-wasm.js"></script>
<script>
  var _originalInitSqlJs = initSqlJs;
  initSqlJs = function(config) {
    config = config || {};
    config.locateFile = function(file) {
      return 'https://cdn.jsdelivr.net/npm/sql.js@1.8.0/dist/' + file;
    };
    return _originalInitSqlJs(config);
  };
</script>
```

## Vercel 배포

### 자동 배포 (GitHub 연동)
GitHub `master` 브랜치에 푸시하면 Vercel이 자동으로 빌드 및 배포함.

### 배포 URL
- https://260301dailyapp.vercel.app

### 빌드 설정
- **vercel.json**: 빌드 커맨드, 출력 디렉토리, SPA 리라이트 설정
- **vercel-build.sh**: Flutter SDK 다운로드 → `flutter build web --release`

### 수동 배포
```bash
npx vercel --prod
```
