# 트러블슈팅

## 해결된 이슈 목록

### 1. Vercel 빌드 실패 - git safe.directory
**증상**: Vercel에서 Flutter SDK 사용 시 git 에러
**원인**: Vercel 빌드 환경에서 git safe.directory 설정 필요
**해결**:
```bash
# vercel-build.sh에 추가
git config --global --add safe.directory /vercel/path0/flutter
git config --global --add safe.directory /vercel/path0
```

### 2. 웹에서 앱 안 보임 - dart:io 크래시
**증상**: Vercel 배포 후 브라우저에서 빈 화면
**원인**: `app_database.dart`에서 `dart:io` 직접 import (웹에서 사용 불가)
**해결**: 플랫폼별 조건부 import 구조로 변경
```
connection/connection.dart → export 'native.dart' if (dart.library.html) 'web.dart'
```

### 3. sql.js 라이브러리 없음
**증상**: `Unsupported operation: Could not access the sql.js javascript library`
**원인**: Drift의 `WebDatabase`가 sql.js에 의존하지만 웹 빌드에 포함 안 됨
**해결**: `web/index.html`에 sql.js CDN 스크립트 추가 (defer 없이 동기 로드)

### 4. WebAssembly magic word 오류
**증상**: `CompileError: WebAssembly.instantiate(): expected magic word 00 61 73 6d, found 3c 21 44 4f`
**원인**: sql-wasm.js가 `.wasm` 파일을 앱 도메인에서 찾으려 함 → HTML 404 반환
**해결**: `initSqlJs` 함수를 패치하여 `locateFile`로 CDN 경로 지정
```javascript
var _originalInitSqlJs = initSqlJs;
initSqlJs = function(config) {
  config = config || {};
  config.locateFile = function(file) {
    return 'https://cdn.jsdelivr.net/npm/sql.js@1.8.0/dist/' + file;
  };
  return _originalInitSqlJs(config);
};
```

### 5. APK 설치 시 다른 앱과 충돌
**증상**: 기존 앱(바이브코딩) 업데이트 팝업 발생
**원인**: 같은 패키지명(`com.dailyapp.daily_app`) 사용
**해결**: 패키지명을 `com.kjs369.dailyhabit`으로 변경
- `android/app/build.gradle.kts` - namespace, applicationId
- `android/app/src/main/kotlin/` - 디렉토리 경로 + package 선언

### 6. Deprecated API - color.value
**증상**: `color.value` deprecated 경고
**해결**: `color.toARGB32()`로 변경

### 7. Drift Value 클래스 충돌
**증상**: `Value` 클래스를 인식하지 못함
**해결**: `import 'package:drift/drift.dart' hide Column;` (Flutter의 Column과 충돌 방지)
